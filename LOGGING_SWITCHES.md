# Переключатели логирования модулей

## Обзор

В систему логирования WPNCollector добавлены переключатели для включения/выключения логов отдельных модулей. Это позволяет гибко управлять объемом логирования и отключать ненужные модули для отладки.

## Настройка

Переключатели настраиваются в файле `config.py` в секции `logging.modules`:

```python
'logging': {
    # ... другие настройки логирования ...
    'modules': {                    # Переключатели логирования для модулей
        'google': True,             # Включить/выключить логи для Google модуля
        'close': True               # Включить/выключить логи для Close модуля
    }
}
```

## Поддерживаемые модули

### Google модуль
- **Имена логгеров**: `GoogleDrive`, `Google`
- **Переключатель**: `google`
- **Описание**: Логирование модуля загрузки файлов в Google Drive

### Close модуль  
- **Имена логгеров**: `CloseTables`, `Close`
- **Переключатель**: `close`
- **Описание**: Логирование модуля закрытия покерных таблиц

### Другие модули
- **Имена логгеров**: любые другие (например, `MainProcess`, `OpenTables`)
- **Переключатель**: не требуется
- **Описание**: Логирование включено по умолчанию для всех остальных модулей

## Использование

### Включение/выключение модулей

```python
# В config.py
CONFIG = {
    'logging': {
        'modules': {
            'google': False,  # Отключить логи Google модуля
            'close': True     # Включить логи Close модуля
        }
    }
}
```

### Изменение настроек на лету

```python
from config import CONFIG

# Отключить Google модуль
CONFIG['logging']['modules']['google'] = False

# Включить Close модуль
CONFIG['logging']['modules']['close'] = True
```

### Проверка статуса модуля

```python
from logger import get_logger

logger = get_logger('GoogleDrive')
# Если модуль отключен, сообщения не будут выводиться
logger.info("Это сообщение может не появиться")
```

## Поведение при отключении

Когда модуль отключен:
- **Консольный вывод**: сообщения не выводятся в консоль
- **Файловый вывод**: сообщения не записываются в файлы логов
- **Производительность**: минимальное влияние на производительность
- **Ошибки**: критические ошибки все равно могут выводиться для отладки

## Примеры использования

### Отладка только основного процесса

```python
# В config.py
'modules': {
    'google': False,  # Отключить Google
    'close': False    # Отключить Close
}
```

### Отладка только Google модуля

```python
# В config.py  
'modules': {
    'google': True,   # Включить Google
    'close': False    # Отключить Close
}
```

### Полное логирование (по умолчанию)

```python
# В config.py
'modules': {
    'google': True,   # Включить Google
    'close': True     # Включить Close
}
```

## Тестирование

Для тестирования переключателей используйте скрипт `test_logging_switches.py`:

```bash
python test_logging_switches.py
```

Скрипт покажет:
- Текущие настройки переключателей
- Тестирование всех модулей
- Демонстрацию работы переключения на лету

## Совместимость

- Переключатели работают с существующей системой логирования
- Обратная совместимость: если переключатели не настроены, все модули работают как раньше
- Работает как с `PrintLogger`, так и с `setup_logger`
